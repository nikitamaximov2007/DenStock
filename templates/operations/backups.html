Как восстановить на новом сервере (только CLI)

Web-восстановление специально отсутствует: оно может затереть текущую БД/media и оборвать сессии. Restore — только через CLI под --yes.

  1. Поднять новый VPS; установить Docker и Git.
  2. git clone проекта.
  3. Создать .env (секреты — заново, не из бэкапа).
  4. docker compose up -d --build.
  5. Положить бэкап в backups/<run_id>/.
  6. docker compose exec web python manage.py restore_db backups/<run_id>/db.dump --yes
  7. docker compose exec web python manage.py restore_media backups/<run_id>/media.tar.gz --yes
  8. docker compose exec web python manage.py ops_check
  9. Проверить: вход, фото (/media/), справочники/остатки/движения.
