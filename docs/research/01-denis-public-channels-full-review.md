## 2. Executive summary

> Выводы ниже основаны только на собранных источниках.

Денис (**канал «Pro-BRP» / probrp1**, YouTube **«Денис Абударов PRO-STOR»**) — **ремонт,
переборка двигателей и обслуживание техники BRP** (снегоходы и водная техника), плюс
**продажа/пригон техники**. Подтверждено прямо из названий видео и текстов постов:

- **Двигатели Rotax** — сквозная тема: серии видео «Rotax 1004. Возврат» (напр. `4AexR3zmI9Q`),
  «900 АСЕ застучал… N-я серия» (18 частей, напр. `ImIOFktB58I`), «Summit Expert 850 сгорело
  магнето» (`DLyINMCZvD4`); посты «Renegade 1200 с движком Rotax 1203» (2296/2295), «Rotax 849
  г. Чебоксары» (2286–2289), «Rotax 552 г. Казань» (2280/2287), «Валы с Rotax 582» (2283). ①②
- **Водная техника (Sea-Doo)**: «Купил RXP-X 300 на аукционе… гидроцикл… с разобранным
  двигателем за 3350$» (`kz7nsf1DHF4`), пост «RXP 255 10 лет простоя» (2285), «116 км по воде»
  (2294). ①
- **Продажа/пригон**: пост 2298 — «Квадроциклы в свободной в г. Бишкек — Outlander XMR …26500$,
  Outlander Backcountry …27800$»; RXP-X куплен на **аукционе в США**. ①
- **Обслуживание (ТО)**: пост 2290 «Снегоход с Новосибирска на ТО». ①
- **География клиентов**: Пермь (пермь — 113 упоминаний; «Пермский COMMANDER» 2291), плюс
  Казань, Чебоксары, Новосибирск, Москва, Бишкек, Киржач (посты). ②
- **Клиентские вопросы** (агрегированно по комментариям): как связаться (~234), сколько стоит/

подбор/заказ деталей → переборка → возврат», плюс **продажа/пригон** аппаратов. Ключевые
запросы — быстро ответить «есть/сколько/цена/подходит ли/где».

## 8. Складские сценарии для DenisStock

> Переведено из подтверждённых данных в рабочие сценарии.

1. **«Есть ли деталь?»** — клиент/мастер спрашивает наличие узла под конкретный мотор (наличие
   ~60+ вопросов). → быстрый поиск по остаткам.
2. **«Под какую модель?»** — подбор запчасти под Rotax 900 ACE / 1004 / VK540 / RXP-X (совмест.
   ~67+15). → поиск по `PartCompatibility`/модели.
3. **«Подходит ли / встанет?»** — проверка совместимости узла (коллектор, магнето, поршневая). →
   карточка детали с совместимостью.
4. **«Сколько стоит?»** — цена запчасти/работы (~135+). → цена в карточке (с учётом прав на
   себестоимость).
5. **Состояние детали** — новая / б/у / с донора / после расточки-никасила (переборка, «RXP 255
   10 лет простоя»). → статусы/пометки экземпляра, фото.
6. **Партии и себестоимость** — пригон/закуп из США (курс $, аукцион). → `Batch` + landed cost.
7. **Фото деталей** — канал построен на фото/видео узлов (магнето, коллекторы, валы). → фото
   `PartType`/`PartItem`.
8. **Ремонтный поток по аппарату** — «привезли из города → дефектовка → детали → переборка →
   возврат». → `RepairOrder` + выдача деталей + `Reservation` под конкретный аппарат.

## 9. Что уже хорошо покрыто текущей системой DenisStock

> Про существующие сущности приложения (факты кода), привязанные к найденным потребностям.

| Потребность (из источников) | Существующая сущность |
|---|---|
| Каталог видов деталей, коды/OEM | `PartType`, `PartNumber` |
| Совместимость с моделью техники | `PartCompatibility` (+ `VehicleType/Make/Model`) |
| Серийные/дорогие узлы (двигатель, коленвал, магнето) | `PartItem` (статус, серийник, место) |
| Расходники/ремкомплекты (масло, сальники, подшипники) | `StockLot` |
| Пригон/закуп партиями + себестоимость (US-аукцион) | `Batch`/`BatchLine` + landed cost |
| Где лежит деталь | `StorageLocation` |
| Приход/расход/движения | `StockMovement`, `StockBalance` |
| Бронь под конкретный аппарат/клиента | `Reservation` |
| Продажа техники/деталей | `Sale` |
| Ремонтный поток и выдача деталей | `RepairOrder` |
| Возвраты | `Returns` |
| Быстрые ответы «есть/сколько/где» | `Reports`, поиск |
| Фотофиксация состояния | `PartTypeImage` / `PartItemImage` |

Вывод: доменная модель **уже соответствует** реальному потоку. Дальнейшее — наполнение
справочников и настройка под терминологию (§10–§11).
