# Раздел 4. Интерфейс и сценарии (складской, операционный)

**Статус:** на согласовании (2026-06-24)

## 4.0. Принципы интерфейса

Интерфейс **операционный, не декоративный**. Главная цель — чтобы Денис или
кладовщик быстро ответили клиенту, приняли партию, нашли/переместили/продали
деталь, провели инвентаризацию.

- **Крупные кнопки** основных действий, пригодны для пальца (планшет/телефон).
- **Минимум текста**, крупные числа. **Статусы отображаются цветовым блоком +
  текстом + иконкой, а не одним только цветом/эмодзи** (ТЗ раздел 23):
  - зелёный блок + иконка + текст **«Успешно»**;
  - красный блок + иконка + текст **«Ошибка»**;
  - жёлтый блок + иконка + текст **«Требуется действие»**.

  *Эмодзи 🟢🔴🟡 в этом документе — только условное обозначение, не элемент
  реального UI.*
- **Мало шагов** на действие; основные сценарии — за 1–3 шага.
- **Сканер-первый**: на складских экранах постоянное поле сканирования с
  автофокусом; Enter запускает действие; быстрый возврат к следующему скану.
- **Адаптивность**: десктоп — основной, планшет/телефон — поддерживаются.
- **Роль-зависимость**: финансовые данные (себестоимость, прибыль, цены)
  скрыты для ролей без доступа.
- Русский язык, рубль.

**Оболочка.** Слева — навигация по разделам (фильтруется ролью); сверху —
глобальный поиск + глобальное поле сканера + пользователь/роль; центр — контент.
На операционных экранах поле сканера закреплено вверху. Есть переключатель
**«Режим сканера»** — увеличивает элементы, прячет лишнее, оставляет цикл
скан → действие.

---

## 4.1. Главная панель

Плитки с крупными числами (ТЗ раздел 24): всего запчастей, доступно, стоимость
остатков, продажи сегодня, продажи за месяц, прибыль за период (выбор периода).
Списки-виджеты: низкий остаток, активные резервы, новые партии, последние
движения, неопознанные детали, обнаруженные проблемы.

Крупные кнопки быстрых действий: **«Найти деталь»**, **«Принять партию»**,
**«Продажа»**, **«Перемещение»**, **«Инвентаризация»**.

Финансовые плитки (стоимость, прибыль) скрыты для роли «Кладовщик».

---

## 4.2. Быстрый поиск детали (во время звонка) — ключевой экран

Большое поле поиска с **автофокусом**, принимает и текст, и скан. Живой результат
по мере ввода (HTMX, debounce). Поиск по всем полям ТЗ раздела 16 (название/часть,
OEM, артикул, штрихкод, внутр. номер, серийный, производитель, категория,
техника/марка/модель/год, аналог, партия, место).

Каждая строка результата сразу показывает: фото-миниатюру, название, OEM,
**доступно** (крупно, цветом), всего, резерв, цену продажи, **местоположение**
(ячейки), партии. Клавиши ↑↓ + Enter; клик/Enter → карточка детали.

Сценарий: во время звонка ввести название/номер → мгновенно видно «есть N штук,
ячейка A-03-02, цена X ₽».

---

## 4.3. Карточка детали

- **Шапка:** название, фото, OEM, режим учёта, статус активности; кнопки
  **«Продать»**, **«Зарезервировать»**, **«Переместить»**, **«Списать»**,
  **«Печать этикетки»**.
- **Наличие:** для количественных — таблица по партиям и ячейкам (партия, ячейка,
  кол-во, доступно, резерв, себестоимость); для поштучных — список экземпляров
  (внутр. номер, серийный, статус, ячейка, себестоимость).
- **Цены:** рекомендуемая, минимальная, последние цены продаж.
- **Совместимость:** техника/марки/модели/годы.
- **Номера и штрихкоды:** OEM, артикулы, аналоги, заводские штрихкоды.
- **История:** движения по детали.

Себестоимость и закупочные данные скрыты для ролей без финансового доступа.

---

## 4.4. Экран поступления партии

Сценарий ТЗ раздела 11.

1. Выбрать/создать партию (шапка: поставщик, номера заказа/счёта, валюта, курс,
   расходы: товар/доставка/таможня/комиссии).
2. Поле сканера: скан заводского штрихкода → поиск детали в каталоге.
   - найдена → карточка;
   - не найдена → **«Создать карточку»** (быстрая форма).
3. Указать количество и **базовую закупочную стоимость в ₽**.
4. Поштучно — регистрация экземпляров (ввод/скан серийных номеров, система
   присваивает внутренние номера `DS-…`); количественно — просто количество.
5. Выбрать/сканировать ячейку (или зону приёмки).
6. Строка попадает в **накопительный список приёмки** (`BatchLine`).
7. Кнопки статусов: «Принять» (завершить приёмку) → **«Рассчитать
   себестоимость»** (распределить накладные) → партия становится **доступной к
   продаже**.

Экран сканер-ориентированный, крупные кнопки, бегущий итог по партии.

---

## 4.5. Экран сканирования (универсальный режим сканера)

Отдельный полноэкранный режим. Большое поле скана, всегда в фокусе.
Скан любого кода → система **определяет тип** (деталь / экземпляр / партия /
ячейка) → показывает карточку результата и контекстные действия.

- Обратная связь блоком (цвет + текст + иконка): «Успешно» / «Ошибка» /
  «Требуется действие».
- Защита от двойного скана (игнор одинакового кода в пределах ~1–1.5 с).
- После действия — автофокус обратно в поле для следующего скана.

Это и есть «режим сканера», который можно включать на любом операционном экране.

---

## 4.6. Экран размещения по ячейкам

Для деталей из зоны приёмки или новых поступлений.
Скан экземпляра/детали → скан целевой ячейки → подтвердить. Для количественных —
скан детали + ячейка + количество. **Проверка `storage_allowed`** (нельзя
положить в служебное/неконечное место). Накопительный список размещённого,
крупная кнопка подтверждения, быстрый цикл.

---

## 4.7. Экран перемещения детали

Сценарий ТЗ раздела 13: скан детали/экземпляра → скан новой ячейки → подтвердить.
Для количественных — запрос количества. Показывает старое → новое место,
комментарий. Создаёт движение `MOVE` (нельзя «просто поменять поле»). Быстрый
цикл для серии перемещений.

---

## 4.8. Экран продажи

ТЗ раздел 18 + логика 3.5.

- Добавление позиций поиском/сканом. Количественно — **лот по FIFO** подставлен,
  можно сменить; количество; цена (по умолчанию рекомендуемая, **ниже
  минимальной — предупреждение**); скидка. Поштучно — конкретный экземпляр.
- **Корзина** с суммами по позициям и итогом; прибыль показывается по роли.
- Клиент, телефон, способ оплаты, общая скидка.
- Крупная **«Провести продажу»** → транзакция с проверкой остатка.
- Партия без зафиксированной себестоимости — блокировка с сообщением.

---

## 4.9. Экран установки в ремонтируемую технику

Как продажа, но: **ремонтируемая техника**, **мастер**, цена клиенту
(`price_charged`). Чётко помечен как другой документ. Та же корзина и
транзакция, движение `INSTALL`. В аналитике — отдельная категория.

---

## 4.10. Экран резервирования

- Создание: поиск/скан детали, количество, опц. конкретный экземпляр/партия,
  клиент + телефон, срок (`expires_at`), комментарий. Показывает доступность.
  Крупная **«Зарезервировать»**.
- Список активных резервов: статус, срок; кнопки **«Преобразовать в продажу»**,
  **«Отменить»**. Зарезервированное не показывается доступным другим.

---

## 4.11. Экран возврата

- Найти исходную продажу (номер / клиент / скан).
- Выбрать позиции для возврата, количество, целевую ячейку (по умолчанию
  исходная), состояние.
- Если деталь повреждена — маршрут на **списание** вместо возврата на склад.
- Крупная **«Оформить возврат»** → движение `RETURN`, восстановление с исходной
  партией и себестоимостью.

---

## 4.12. Экран списания

- Скан/выбор детали или экземпляра, количество, **причина** (список),
  **обязательный комментарий**.
- Статус `PENDING` → подтверждение ролью «Руководитель»/«Администратор»
  (`CONFIRMED`).
- Стоимость потери (себестоимость) — по роли. Крупная кнопка, цикл сканера.

---

## 4.13. Экран инвентаризации

- Создать инвентаризацию, выбрать область (склад / зона / ячейка).
- **Режим подсчёта (сканер-первый):** скан ячейки → скан деталей в ней →
  количества накапливаются.
- Сравнение факт/система: недостачи и излишки цветом.
- Подтверждение корректировок (роль) → движения `INV_SURPLUS`/`INV_SHORTAGE`.
- Сохранение отчёта. Крупный прогресс, быстрый цикл сканирования.

---

## 4.14. Экран статистики продаж за период

- **Период:** сегодня / вчера / неделя / месяц / квартал / год / произвольные даты.
- **Показатели:** сумма продаж, себестоимость проданного, валовая прибыль,
  кол-во проданных деталей, кол-во продаж, средний чек, средняя прибыль с продажи.
- **Разрезы:** самые продаваемые, самые прибыльные, по категориям, типам техники,
  производителям, партиям, сотрудникам.
- **Подробная таблица** продаж с фильтрами: период, категория, тип техники,
  деталь, партия, сотрудник, способ оплаты, поставщик.
- **Экспорт:** Excel, CSV, PDF.
- Экран финансовый — доступ ролям «Руководитель»/«Администратор».

---

## 4.15. Экран партий

Список партий: статус, поставщик, даты, итог в ₽, признак «доступна к продаже».
Фильтры по статусу/поставщику. Открытие партии → строки поступления, расходы,
документы, кнопка **«Рассчитать себестоимость»**, переходы статусов. Создание/
редактирование партии (расходы и курс редактируемы только до фиксации).

---

## 4.16. Экран структуры склада

Дерево мест: склад → зона → стеллаж → секция → полка → ячейка. Добавление/
редактирование узлов: название, код, штрихкод, `storage_allowed`, вместимость,
статус. Клик по ячейке → её содержимое. Кнопка печати этикеток мест.
Управляется администратором.

---

## 4.17. Экран печати этикеток

Согласно решению: генерация внутренних кодов + **PDF-лист этикеток** (сетка на
листе A4, обычный принтер; принтер этикеток не требуется).

- Выбор объектов: детали / экземпляры / ячейки / партии.
- Выбор шаблона/размера этикетки, количество на объект.
- Предпросмотр → **«Сформировать PDF»** → печать.

## 4.19. UX со сканером (детально)

- **USB-сканер = клавиатура**: набирает код и шлёт Enter.
- **Постоянное поле сканирования** на складских экранах, **автофокус**;
  возвращает фокус после каждого действия и при клике мимо.
- **Enter** запускает резолв и нужное действие.
- **Определение типа кода** единой точкой, по порядку:
  внутренний номер (`DS-…`) → код/штрихкод ячейки → серийный номер →
  заводской штрихкод → OEM/артикул. Возвращает тип объекта (деталь / экземпляр /
  партия / ячейка) и сам объект.
- **Состояния** крупно и по центру, каждое — цветовой блок + текст + иконка:
  «Успешно» / «Ошибка» / «Требуется действие» (например, код неоднозначен —
  выбрать).
- **Защита от двойного скана**: одинаковый код в пределах ~1–1.5 с игнорируется.
- **Крупные кнопки**, **быстрый возврат** к следующему сканированию после успеха.
- **Режим сканера** — переключатель, увеличивающий элементы и сводящий экран к
  циклу скан → действие.
