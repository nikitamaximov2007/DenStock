# Preset «mototech» — стартовые справочники (опционально)

Опциональный **стартовый набор справочников** для ниши ремонта мототехники (снегоходы,
гидроциклы, двигатели). Реализует план [33](../plans/33-research-based-directory-preset.md).

**Важно:**
- DenisStock остаётся **универсальной** системой. Preset — это просто данные для существующих
  справочников, а **не** изменение архитектуры и **не** «BRPStock».
- Preset наполняет **только справочники** и **не создаёт остатки** (никаких Batch, StockLot,
  PartItem, StockMovement, StockBalance).
- Данные основаны на подтверждённых терминах из
  [docs/research/01](../research/01-denis-public-channels-full-review.md) и
  [02](../research/02-denis-business-summary-for-owner.md). `research_inputs/` не используется и
  не коммитится.
- Записи — обычные справочники: их можно **менять и удалять** после установки.

## Что входит (data/presets/mototech/*.json)

| Файл | Модель | Записи |
|---|---|---|
| `manufacturers.json` | `catalog.Manufacturer` | BRP, Rotax, Yamaha, Mercury, MerCruiser |
| `vehicle_types.json` | `catalog.VehicleType` | Снегоход, Квадроцикл (уже сидятся миграцией), Гидроцикл (новый) |
| `vehicle_makes.json` | `catalog.VehicleMake` | Ski-Doo, Lynx, Yamaha (снегоход), Sea-Doo (гидроцикл), Can-Am (квадроцикл) |
| `vehicle_models.json` | `catalog.VehicleModel` | Summit, Summit Expert 850, Renegade 1200, VK540, RXP-X 300, RXP 255, Outlander XMR, Outlander Backcountry, Commander |
| `categories.json` | `catalog.Category` | Двигатель, Поршневая группа, Коленвал, Цилиндр, Клапаны, Подшипники, Сальники и прокладки, Коллекторы, Магнето и электрика, Вариатор, Масла и жидкости, Гусеницы/склизы/лыжи |
| `part_types.json` | `catalog.PartType` | двигатели Rotax 900 ACE / 1004 / 1203 (serial); магнето/коллектор/вариатор/коленвал (serial); поршневая/клапан/подшипник/сальник/прокладка/масло (bulk) |
| `storage_locations.json` | `warehouse.StorageLocation` | примерные зоны `PRESET-ENG/CPG/ELEC/OIL/TRACK` (образцы — поправьте под свой склад) |
| `suppliers.json` | `suppliers.Supplier` | один **условный демо-поставщик** (не реальные данные) |
| `search_synonyms.json` | — (data-файл) | справочный словарь синонимов; **в поиск не интегрируется** в этом слое |

## Сознательно НЕ входит (отложено)

- **`part_numbers.json` (OEM/каталожные номера)** — в research **нет подтверждённых** OEM-кодов.
  Коды двигателей (900 ACE, 1004, 1203) отражены в названиях `PartType`. Не выдумываем номера.
- **`compatibility.json` (`PartCompatibility`)** — отложено, чтобы первый слой был минимальным и
  безопасным. Есть очевидные связи из research (напр. магнето ↔ Summit Expert 850; двигатель
  Rotax 1203 ↔ Renegade 1200) — добавим отдельным слоем.
- **Лодочные моторы / катера как отдельный вид** — в research упоминаются реже (Mercury/MerCruiser
  как производители внесены, но `VehicleType` «Лодочный мотор» — вопрос к Денису).
- **Любые остатки, партии, цены, клиенты** — не создаются принципиально.

## Как запустить

```bash
# Показать план без записи в БД:
python manage.py import_preset --preset mototech --dry-run

# Применить (наполнить справочники):
python manage.py import_preset --preset mototech
```

Команда идемпотентна: повторный запуск **не создаёт дублей** (get_or_create по естественным
ключам). `--dry-run` откатывает все операции и ничего не сохраняет.

## Как проверить результат

- В админке/интерфейсе появились виды деталей (напр. «Двигатель Rotax 900 ACE»), марки/модели
  техники, категории и зоны `PRESET-*`.
- Остатки **не** изменились (движений/лотов/экземпляров не создано).
- Сводка команды показывает `создано / существует / отложено` по каждой секции.

## Ограничения

- Preset — стартовое **заполнение справочников**, а не реальные остатки. Реальный склад Денис
  заводит сам (или отдельным слоем импорта).
- Зоны хранения и демо-поставщик — образцы, замените под свою практику.
- `search_synonyms.json` пока не влияет на поиск (интеграция — отдельный план).
