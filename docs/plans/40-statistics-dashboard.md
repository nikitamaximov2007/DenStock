# План 40 (Layer 27) - Раздел «Статистика»: read-only срез склада

**Статус:** реализовано в Layer 27. Активирован серый пункт меню «Статистика»
(рекомендация из [плана 39](39-full-functionality-qa-and-readiness.md), §7-8).

---

## 1. Чем «Статистика» отличается от «Отчётов»

| | Отчёты (`/reports/`) | Статистика (`/statistics/`) |
|---|---|---|
| Вопрос | «Что произошло за период?» | «Что лежит на складе сейчас и на что смотреть?» |
| Основа | проведённые документы (продажи, ремонт, возвраты, списания, корректировки) | текущие остатки, landed cost, резервы + подсказки |
| Период | обязателен (все блоки периодные) | влияет только на «ходовые», «активность» и порог «залежавшихся»; стоимость склада и низкие остатки - срез сейчас |
| Экспорт | CSV | нет (витрина) |

Расчёты отчётов (`apps/reports/services.py`) не менялись.

## 2. Реализованные метрики

**KPI (срез сейчас):** стоимость склада по себестоимости (landed cost доступных
экземпляров + лотов), потенциальная выручка (доступно x рекомендованная цена, только
где цена задана), видов на складе, доступно единиц, активные резервы, низкие остатки,
позиции без движения, ячеек с товаром.

**Деньги в складе:** стоимость по категориям (top 10), топ деталей по стоимости
остатка (top 10, ссылки на карточки), счётчик «не оценено» (позиции с landed cost 0 -
не искажают сумму, показываются отдельно).

**Низкие остатки:** доступно < `min_stock_level` (поле уже было в PartType); ссылка на
деталь. Та же логика, что в отчёте low-stock (отдельная копия запроса + pk, отчётный
сервис не тронут).

**Залежавшиеся позиции:** есть физический остаток, но нет движений дольше порога
(порог = выбранный период 7/30/90 дней; для «всё время» - 90). Показывает остаток,
стоимость, дату последнего движения или метку «движений не было». Сортировка по
стоимости (замороженные деньги сверху), cap 20 строк.

**Ходовые позиции (за период):** топ 10 по количеству из строк проведённых продаж
(количество, выручка, прибыль). Источник - те же поля SaleLine, что у отчётов.

**Резервы, требующие внимания:** активные резервы (до 10, ближайшие к истечению
первыми): просроченные (badge «просрочен»), истекающие в 7 дней («истекает»).

**Активность склада (за период):** движения по группам (приёмка, перемещения, продажи,
выдачи в ремонт, возвраты, списания, корректировки) + всего.

## 3. Какие данные используются

Только существующие таблицы: `StockBalance` (кэш остатков), `StockLot`, `PartItem`
(landed cost), `StockMovement` (журнал), `Reservation`, `Sale`/`SaleLine`,
`PartType.recommended_price/min_stock_level`, категории. **Новых моделей и миграций
нет.**

## 4. Ограничения (честные)

- Стоимость склада учитывает только ДОСТУПНЫЕ экземпляры/лоты (карантин, ремонт,
  приёмка не оцениваются в «деньги в складе»); позиции без себестоимости показаны
  счётчиком «не оценено».
- Потенциальная выручка считается только по деталям с заданной рекомендованной ценой.
- «Залежавшиеся» смотрят на последнее движение вида детали (не конкретного лота).
- Оборачиваемость как коэффициент не считается (нужна история среднего остатка) -
  отложено; практический заменитель: «ходовые» + «залежавшиеся».
- Топы ограничены (10/20 строк), тяжёлых запросов нет: агрегаты ORM, один проход по
  таблицам, слияние по видам деталей в Python.

## 5. Почему read-only

Статистика - управленческая витрина. Любая запись (пересчёт кэша, авто-отмена
резервов) из витрины - скрытый побочный эффект и риск для данных. Тест
`test_statistics_page_is_readonly` фиксирует: GET не меняет движения и остатки.

## 6. Доступ

Гейт `can_view_finance` (как у серого пункта меню ранее): Администратор, Руководитель,
Наблюдатель. Продавец и Кладовщик получают 403 (страница целиком денежная, поэтому
отдельного скрытия сумм как в отчётах не требуется).
