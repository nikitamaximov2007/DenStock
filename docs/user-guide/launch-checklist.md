# DenStock: чеклист перед реальным запуском

Пройти перед тем, как перевести склад Дениса на ежедневную работу в системе.

## Доступы
- [ ] Создан личный пользователь для каждого сотрудника (не общий логин)
- [ ] Роли выданы по работе: кладовщик, продавец, руководитель
- [ ] Администратор один-два человека, пароль надёжный
- [ ] Каждый сотрудник смог войти под собой

## Справочники и структура
- [ ] Категории отражают реальный ассортимент (без «мусорных»)
- [ ] Поставщики заведены
- [ ] Ячейки созданы, коды понятные (A-01, B-03)
- [ ] Этикетки ячеек распечатаны и наклеены на полки

## Каталог
- [ ] Основные ходовые детали заведены
- [ ] У ходовых деталей заполнены: режим учёта, OEM/штрихкоды,
      рекомендуемая цена, минимальный остаток
- [ ] Поиск находит детали по номеру и названию
- [ ] Сканер распознаёт этикетки деталей и ячеек

## Начальные остатки
- [ ] Фактические остатки заведены поступлениями (с реальной себестоимостью)
- [ ] «Остатки» совпадают с полками (выборочная проверка 10-20 позиций)
- [ ] «Статистика»: стоимость склада выглядит правдоподобно,
      блок «не оценено» пуст или объясним

## Процессы (прогнать по одному разу)
- [ ] Поступление: черновик -> позиции -> провести -> остатки выросли
- [ ] Продажа: черновик -> позиции -> провести -> остаток списался
- [ ] Резерв: создать -> активировать -> «Доступно» уменьшилось -> отменить
- [ ] Ремонт: выдать деталь -> остаток списался
- [ ] Возврат: из продажи -> остаток вернулся
- [ ] Списание: провести -> появилось в отчёте по причине
- [ ] Перемещение: деталь переехала между ячейками
- [ ] Инвентаризация: по одной ячейке, факт введён, корректировка создана

## Аналитика
- [ ] «Отчёты» показывают тестовые продажи за период
- [ ] CSV-экспорт открывается в Excel
- [ ] «Статистика» показывает ходовые и залежавшиеся

## Бэкапы (критично)
- [ ] «Экспорт бэкапа» создаёт бэкап
- [ ] Offsite-статус «отправлено» (настроен rclone и расписание на сервере)
- [ ] Ответственный знает: раз в неделю проверять страницу «Бэкапы»
- [ ] Администратор сервера один раз проверил восстановление из бэкапа
      на тестовой машине (по runbook в docs/operations/)
- [ ] Аварийное веб-восстановление: флаг DENSTOCK_ENABLE_WEB_RESTORE
      ВЫКЛЮЧЕН в обычной работе (включать только на время аварии);
      allowlist содержит только владельца

## Обучение
- [ ] Продавец умеет: искать, резервировать, продавать
- [ ] Кладовщик умеет: поступление, приёмка/перемещение сканером, списание,
      инвентаризация
- [ ] Руководитель умеет: читать отчёты и статистику, заводить детали
- [ ] Инструкция передана команде (docs/user-guide/denstock-user-manual.md)

## Правила команды
- [ ] Всё через систему: никаких «потом занесу»
- [ ] Ошибки исправляем корректирующими операциями, не просьбами
      «поправить базу»
- [ ] Обещал клиенту: поставь резерв со сроком
- [ ] Не знаешь, как сделать: сначала инструкция, потом вопрос руководителю
