- Что сделано:
  - `PartCustomsInfo.ApplicationArea` (TextChoices) - единственный источник
    контролируемых категорий: СНЕГОХОД, КВАДРОЦИКЛ, ГИДРОЦИКЛ, КАТЕР / ЛОДКА,
    АВТОМОБИЛЬ, УНИВЕРСАЛЬНЫЕ ЗАПЧАСТИ. `resolve_customs_application()` в
    services.py берёт значения из того же списка - расхождений между ручным
    select и автоопределением по совместимости больше нет.
  - `application_area`: `default=""` вместо легаси `"МОТО ЗАПЧАСТИ"`,
    `blank=True`, `choices=ApplicationArea.choices`. Свободный текст в UI
    больше не принимается.
  - Приоритет (уже был в services.py, просто закреплён): 1) явное значение
    карточки, 2) `resolve_customs_application()` по `PartCompatibility`,
    3) пусто. Легаси `"МОТО ЗАПЧАСТИ"` трактуется как «не заполнено» и никогда
    не экспортируется.
  - `part_export_data()` возвращает `application_source`
    (manual / compatibility / none) для UI.
  - Новый эндпоинт `actions_customs_application` (POST-only,
    `/inventory/actions/customs/<part_id>/application/`) - быстрое построчное
    сохранение прямо со страницы отчёта. Создаёт `PartCustomsInfo` только по
    явному POST (get_or_create внутри `transaction.atomic`), валидирует
    значение по `ApplicationArea.values`, пишет `updated_by`/`updated_at`.
    Доступ - те же права, что у остальных действий сканера
    (`_require_access`); GET на эту же страницу отчёта ничего не создаёт
    (`build_export_rows` -> `part_export_data` -> `read_customs`, read-only).
  - Страница `/inventory/actions/report/`: новая карточка «Область применения
    для таможни» - таблица ВСЕХ строк экспорта (производитель, точный
    артикул, название, количество, источник, текущее значение, select +
    кнопка «Сохранить» на каждую строку). Счётчики «Готово / не заполнено» и
    предупреждение у кнопки экспорта «Позиций без области применения: N. В
    Excel эти ячейки останутся пустыми.» при N > 0.
  - Существующая детальная форма `actions_customs_edit` (RU-название, веса):
    свободный текст `application_area` заменён на тот же `<select>`; попутно
    исправлен баг, где пустой выбор в этой форме молча записывал легаси
    `"МОТО ЗАПЧАСТИ"` вместо пустой строки.

    добавлено: сама задача разрешает это при риске случайной массовой записи,
    построчного редактора достаточно.
